# Goal

**Query**: `$1` optional issue/PR number or URL. **Context**: `$2` optional priority. **Automation**: `--auto`; add `--auto-merge` to merge the verified PR.

Run `/goal` only in the main agent. Before any state acquisition, establish that this is the main agent; if the harness cannot establish that role, stop and report it without delegation or mutation. Read root `AGENTS.md`, `_common-github-ownership.md`, and rules selected by each phase. Require exactly one open non-roadmap issue. If `$1` is omitted, resolve the active target with `uv run python .agents/scripts/workflow_state.py resolve-active --format json`; otherwise use `$1` as `$TARGET`. Acquire `$TARGET` from the primary checkout before reading or initializing its state:

```bash
PRIMARY=$(dirname "$(git rev-parse --path-format=absolute --git-common-dir)")
TARGET_RESULT=$(cd "$PRIMARY" && uv run python .agents/scripts/resolve-target-worktree.py "$TARGET")
```

Require one returned worktree path for the resolved target; all delegated source-aware phases run from that returned worktree. Re-fetch the open non-roadmap, non-`spec`-labelled issue body and every recorded or branch-matching PR body and metadata. A `spec`-labelled issue cannot be claimed or delivered as an implementation goal. In the returned worktree, initialize an issue target idempotently from fetched canonical metadata with `uv run python .agents/scripts/workflow_state.py init OWNER/REPO#NUMBER --title <title> --url <url> --objective <objective> --format json` before `uv run python .agents/scripts/workflow_state.py show OWNER/REPO#NUMBER --format json`. For a direct PR target, initialize its PR state with `uv run python .agents/scripts/workflow_state.py init-pr OWNER/REPO!NUMBER --title <title> --url <url> --head <head> --format json`, then validate a reused plan with `uv run python .agents/scripts/workflow_state.py validate-plan-head OWNER/REPO!NUMBER --head <head> --format json`; never use issue-only state lookup for a PR target.

Preview the authenticated login and all resolved issue/PR numbers. Without `--auto`, request one complete run authorization covering local worktrees, claims, implementation, commit, push, PR writes, review, and readying; with `--auto`, the invocation is that authorization. After authorization is established, initialize the trace once with `uv run python .agents/scripts/goal_trace.py init OWNER/REPO#NUMBER [--auto] [--commit] [--push] [--pr-create] [--pr-ready] [--merge] [--administrator-merge]`, using the re-fetched linked issue identity for a direct PR target. Pass each flag only when its matching current permission fact is true. Trace configuration is observability metadata, not authorization: it records current permission facts only and never grants permissions; authorization remains the current `/goal` invocation and required explicit permissions. Claim each with `uv run python .agents/scripts/gh.py claim <issue-or-pr-number> --format json` before resuming implementation or delivery. Preserve existing assignees and stop on partial failure. Then resume the recorded phase rather than repeating completed work.

The main agent is orchestration-only: it alone acquires state, authorizes batches, directly dispatches sibling subagents, evaluates delegated evidence, and advances lifecycle state. Each phase agent begins `Read root AGENTS.md first`, reports results or blockers to the main agent, and phase agents do not spawn agents. This workflow requires no nested-agent, background-process, polling, or harness-specific runner capability. A missing, failed, or malformed delegated result blocks the goal without advancing state; the main agent must not perform parent-authored substitute work. After every accepted meaningful delegated result, run `uv run python .agents/scripts/workflow_state.py reset-environment-retry OWNER/REPO#NUMBER --format json`. For an environment failure only, not a failing application test with an available in-scope fix, run `uv run python .agents/scripts/workflow_state.py record-environment-failure OWNER/REPO#NUMBER --fingerprint <stable-fingerprint> --resume-action <exact-safe-resume-action> --format json`; the fifth of five consecutive identical failures blocks. A different fingerprint starts a new count.

For phase transitions, delegation, pause and resume, block, ready, and merge outcomes, validate the complete trace before appending either record: `uv run python .agents/scripts/goal_trace.py validate OWNER/REPO#NUMBER`. Then append one concise log with `uv run python .agents/scripts/goal_trace.py append-log OWNER/REPO#NUMBER --event <transition|pause|resume|reason|continuation> --phase <phase> --summary <concise-safe-summary> [--reason <concise-safe-reason>] [--resume-action <exact-safe-resume-action>]` and one detailed audit with `uv run python .agents/scripts/goal_trace.py append-audit OWNER/REPO#NUMBER --stage <acquire|branch|plan|implement|deliver|review|remediate|validate|verify|ready|merge> --outcome <started|succeeded|blocked|failed|skipped> --detail <detailed-safe-facts> [--reason <concise-safe-reason>] [--resume-action <exact-safe-resume-action>]`. Never record secrets, raw command output, environment values, credentials, tokens, URLs with query strings, or authorization material. If trace validation or either append fails, block without advancing state and report the safe resume point.

1. Branch/worktree: use the acquired returned worktree. The resolver validates and returns the existing head branch or canonical `<type>/<issue-number>-<lower-kebab-slug>` issue branch. For an issue target, record the branch with `uv run python .agents/scripts/workflow_state.py set-branch OWNER/REPO#NUMBER <branch> --base <base> --format json` and run `uv run python .agents/scripts/workflow_state.py transition OWNER/REPO#NUMBER branched --status active --clear-pending-action --format json`.
2. Planning: from the returned worktree, the main agent directly dispatches fresh subagent A as a sibling: `Read root AGENTS.md first, then run @.agents/commands/plan.md OWNER/REPO#NUMBER $2 --auto.` Stop unless A returns valid complete-plan evidence.
3. Implementation: after A returns valid evidence, the main agent directly dispatches distinct fresh subagent B as a sibling: `Read root AGENTS.md first, then run @.agents/commands/implement.md OWNER/REPO#NUMBER $2 --auto.` `/implement` executes the complete plan test-first, commits, pushes, and returns valid implementation evidence.
4. PR delivery: after B returns valid evidence, the main agent directly dispatches distinct fresh subagent C as a sibling: `Read root AGENTS.md first, then run @.agents/commands/create-pr.md OWNER/REPO#NUMBER --auto.` Stop unless C returns valid draft-PR evidence.
5. Baseline review: after C returns valid evidence, run `uv run python .agents/scripts/workflow_state.py transition OWNER/REPO#NUMBER reviewing --status active --clear-pending-action --format json`, then the main agent directly dispatches distinct fresh subagent D as a sibling: `Read root AGENTS.md first, then run @.agents/commands/review.md --auto.` Retain D's task identity. If no OPEN findings remain, the baseline review passes. While mechanically clear, in-scope OPEN findings remain, the main agent continues by directly dispatching fresh subagent E as a sibling: `Read root AGENTS.md first, then run @.agents/commands/review-implement.md --auto.` Then the main agent directly resumes D by task identity for `Read root AGENTS.md first, then run @.agents/commands/review-validate.md --auto.` When D cannot be resumed, the main agent directly dispatches validation to a fresh validator sibling, which records the fallback in its result. After valid validation evidence, the main agent directly dispatches it to fresh subagent F as a sibling for PR delivery: `Read root AGENTS.md first, then run @.agents/commands/create-pr.md OWNER/REPO#NUMBER --auto.` Then directly dispatch a fresh sibling baseline reviewer and repeat until no OPEN findings remain; every subsequent baseline review uses a new subagent rather than a prior reviewer. Do not loop or guess at subjective fixes. Scope expansion, ambiguity, a security or permission gate, a conflict requiring human choice, non-mechanical findings, or invalid delegated evidence block immediately with the exact reason and safe resume point. After valid delegated review evidence returns, record its canonical evidence with `uv run python .agents/scripts/workflow_state.py set-review OWNER/REPO#NUMBER <canonical-report> --state <ACTIVE_OPEN|COMPLETE|CLEAN|ARCHIVED> --archive <canonical-archive> --format json`; omit `--archive` unless the state is `ARCHIVED`.
6. Re-run required verification and inspect every recorded PR, branch, review report, and worktree. Only when checks pass, no OPEN findings remain, each pushed PR head matches its local branch, and the worktree is clean, run `uv run python .agents/scripts/gh.py cmd --format json pr ready <pr-number>`, fetch the PR, and require `isDraft` is `false`. Merge only when `--auto-merge` was supplied and GitHub reports it mergeable: `uv run python .agents/scripts/gh.py cmd --format json pr merge <pr-number> --merge`, then fetch it and require `mergedAt`. Never use an administrator-policy override without separate explicit authorization. Otherwise report `goal_delivered: clean merge-ready PR`. Then run `uv run python .agents/scripts/workflow_state.py transition OWNER/REPO#NUMBER goal_delivered --status goal_delivered --clear-pending-action --format json`.

## Required Context

- Root `AGENTS.md`; issue state; delegated primitive commands; applicable rules and review contracts.

## Mutations

- Confirmed issue/PR ownership claims, delegated planning/implementation/review changes, explicitly permitted commit and push, PR metadata, and ignored workflow state.

## Confirmation

- Without `--auto`, one complete run authorization is required before mutation. `--auto` and inherited authorization cover the complete workflow; `--auto-merge` is required for merging.

## Failure

- Set the current phase blocked and stop on scope expansion, ambiguity, failed delegation/verification, a security or permission gate, conflict requiring human choice, non-mechanical review findings, invalid delegated evidence, dirty final state, or stale PR head. Record the exact reason and safe resume point in `pending_action`. Preserve state for resume; never claim `goal_delivered` early.
